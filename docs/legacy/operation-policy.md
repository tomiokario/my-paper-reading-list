# 論文管理カンバン運用方針

本ドキュメントは、GitHub Projectsを用いた論文読み込みのワークフローと、各ステータス（カラム）の定義についてまとめたものである。

## 1. カンバン構造とステータス定義

本プロジェクトでは以下の5つのステータスを用いて進捗を管理する。

| ステータス | 定義 | 役割 |
| :--- | :--- | :--- |
| **Backlog** | 未処理・インボックス | GASにより自動登録された直後の状態。まだ中身を確認していないもの。 |
| **Pending** | 保留・判断待ち | タイトルや概要だけでは判断がつかない、または後で精査したいもの。 |
| **Ready** | 読む予定 (ToDo) | 選別が完了し、「読む」と意思決定したもの。着手待ちのキュー。 |
| **In Progress** | 読書中 | 現在読んでいる、またはメモをまとめている最中のもの。 |
| **Done** | 完了 | 読み終わり、必要なメモを残してクローズしたもの。 |

---

## 2. 運用ワークフロー

論文は原則として以下のフローで右方向へ移動させる。

### Phase 1: 自動収集 (Entry)
* **GAS → Backlog**
    * Gmailに届いたアラートは自動的にIssue化され、`Backlog` カラムに追加される。
    * この段階では人間は介入しない。

### Phase 2: 選別 (Triage)
`Backlog` に溜まったカードを定期的に確認し、以下の基準で振り分ける。

1.  **不要な場合**
    * タイトルや概要を見て興味がない、関連性が薄いと判断したもの。
    * **アクション**: `Project`から外して`Close`する。もしくは、Issueを削除する。
2.  **迷う場合**
    * 判断に迷うもの。
    * **アクション**: `Pending` に移動する。
3.  **読む場合**
    * 明確に興味があり、読む必要があるもの。
    * **アクション**: `Ready` に移動する。

### Phase 3: 読書と記録 (Execution)
1.  **着手**
    * `Ready` から読む論文を選び、`In Progress` に移動する。
    * リンク（Source URL / OA URL）から論文にアクセスする。
2.  **記録**
    * 気になった箇所、アイデア、批判的検討などをIssueの **Comment** に追記する。

### Phase 4: 完了 (Completion)
1.  **クローズ**
    * 読み終わったら、Issueを **Close** する。
    * Closeされたアイテムは自動的に `Done` カラムへ移動される。
